Выбор файловой системы

В пункте 1 я определился с выбором операционной системе, ей стала актуальная Windows. Далее, необходимо разметить диск и определиться с файловой системой. Изначально при установке системы Windows диск форматируется в файловой системе NTFS, что является необходимым условием для установки Windows 11.

Схемой разделов выбрана MBR, так как она полностью соответствует поставленным задачам в конкретной системе. А именно: в системе не понадобится более 4 разделов, файловое пространство будет объемом 60 Гб. Решающим звеном в выборе схемы разделов стало то, что система будет установлена на виртуальную машину через Oracle VirtualBox, который не поддерживает интерфейс UEFI.

После установки системы и необходимого программного обеспечения имеем 1 раздел жесткого диска на 63,3 Гб из которых 40,3 Гб свободны.

Для удобства использования я разделю файловое пространство на разделы. Будет 2 раздела: 1 раздел для хранения системы и системного по, 2 раздел для хранения всех остальных личных файлов и программ.

При создании второго раздела необходимо будет выбрать файловую систему: NTFS или FAT32. FAT32 является старой файловой системой, которая была введена при Windows 95. Главным ее недостатком является ограничению по размеру файлов в 4 Гб и отсутствие безопасности. NTFS – современная файловая система, которую Windows использует по умолчанию. NTFS обеспечивает наилучшую безопасность: шифрование, права доступа к файлам, журнал изменений. Несмотря на свои преимущества, недостатком NTFS является совместимость, файловая система работает только с Windows и на чтение с macOS. Подводя итоги по выбору файловой системы для создания второго раздела, можно сказать, что выбор файловой системы NTFS будет наилучшим решением.

Таким образом, файловое пространство системы разделено на два раздела в системе NTFS.
